    DenseNet

    是基于ResNet的一种变体(或许所有带跳远连接的都算是其变体吧)， 但是其block十分独特， 整个网络层仅由3-4个block构成。
    虽然block少，但是对于跳远连接来说可以点都不少，对于一个有L层的block来说，足足有(L×(L + 1)) / 2 个跳远连接(wtf?)

    DenseNet构成及其大致原理

    首先我们先简单聊一下DenseNet的结构:

    • 对于一个block，作者采用了类似于Inception的方式，对于每个跳远连接，我们不像ResNet那样进行数值相加，而是对于feature-map的直接叠加，然后再进行1×1的卷积进行降维，ResNet的直接数值相加阻碍了信息之间的流通，也就是作者说的information flow。
    • 然后对于每一个3×3卷积，我们限定他的维度为k，然后我们进行叠加的时候我们会发现，假如他再第L层，那么他就会有(L * k + k0)个feature-map，这个数量进行卷积的计算量我们是承受不住的，所以叠加之后，我们就用上刚才说的降维了，这里我们定义1×1的卷积,然后设定特征图的数量为4k。
    • 然后我们对于block中的每一层(except pooling layer), 在其之后都加上一个复合函数H(l)，这个函数由三种元素构成(BN, Relu, conv),作者称之为DenseNet-B。
    • 我们在每一个block处理完之后，为了提高我们的模型简洁性，我们在过滤层减少了输入的(上一层的输出)feature-map的数量，我们需要定义一个阙值theta，然后我们取的特征图是floor(theta * 特征图的数量)，这一阶段我们叫做DenseNet-C，我们与上一步合起来叫做DenseNet-BC
    • 在一个block块中，我们要保证维度始终是相同的，才能使得最后叠加起来，但是我们还要进行特征聚合，所以我们只有在连接两个block的时候才会用到pooling layer

    大致原理：
    DenseNet 提高了信息流再各个块之间流动，使得起点到终点非常近，可以类似于直接相连，使得梯度不会消失，数据信息流通的相当好，所以才会准确率高。
    DenseNet-C的处理效果: 比如我们drop掉一个特别重要的feature-map，使得特征丢失，我们在反馈的时候会自动的让别的feature-map学习到这个特征，增强了学习的适应能力。
